What is umask?

Umask determines the default permissions for newly created files and directories by subtracting from the maximum permissions (666 for files, 777 for directories).

Current Umask:
0022
New Files (666 - umask):
644 (rw-r--r--)
New Directories (777 - umask):
755 (rwxr-xr-x)
View Current Umask:
umask
Set Umask (session):
umask 0022
Make Permanent (~/.bashrc):
echo "umask 0022" >> ~/.bashrc

üìñ Linux Permissions Quick Reference

chmod - Change File Permissions

Syntax: chmod [options] mode file

Common Options:

  • -R - Recursive (apply to directories and their contents)
  • -v - Verbose (show files being processed)
  • --reference=FILE - Use FILE's mode instead of MODE
chmod 755 script.sh
chmod u+x,g-w file.txt

chown - Change File Owner

Syntax: chown [options] owner[:group] file

chown john file.txt
chown john:developers /var/www
chown -R www-data:www-data /var/www/html

chgrp - Change Group Ownership

Syntax: chgrp [options] group file

chgrp developers project/

setfacl - Set File Access Control Lists

Syntax: setfacl [options] acl_spec file

Common Options:

  • -m - Modify ACL
  • -x - Remove ACL entry
  • -b - Remove all ACL entries
  • -d - Set default ACL (for directories)
  • -R - Recursive
setfacl -m u:john:rwx file.txt
setfacl -m g:developers:rx directory/
setfacl -d -m u:john:rwx /shared/

getfacl - Display File ACLs

Syntax: getfacl file

getfacl file.txt

umask - Set Default Permissions Mask

Syntax: umask [mask]

umask
umask 0022

chattr - Change File Attributes

Syntax: chattr [options] [+|-|=attributes] file

Common Attributes:

  • i - Immutable (cannot be modified, deleted, or renamed)
  • a - Append only (can only append data)
  • d - No dump (excluded from backup)
  • s - Secure deletion (overwrite with zeros)

Common Options:

  • + - Add attributes
  • - - Remove attributes
  • = - Set exact attributes
  • -R - Recursive
chattr +i important.conf
chattr -i important.conf

lsattr - List File Attributes

Syntax: lsattr [options] file

lsattr important.conf

visudo - Edit Sudoers File Safely

Syntax: visudo [options]

Common Options:

  • -c - Check syntax without editing
  • -f FILE - Edit specific file (e.g., /etc/sudoers.d/custom)
sudo visudo
sudo visudo -c
sudo visudo -f /etc/sudoers.d/custom

About Permissions:

r (read): View file content, list directory contents

w (write): Modify file content, create/delete files in a directory

x (execute): Execute a file, enter a directory

SetUID (s): File runs with owner's privileges (not user who executes it)

SetGID (s): File runs with group's privileges; on directories, new files inherit the directory's group

Sticky Bit (t): On directories, only file owner, directory owner, or root can delete/rename files

Note on Case (s/S, t/T):
Lowercase (s, t): Execute permission (x) is present.
Uppercase (S, T): Execute permission is missing (ineffective special bit).

About Ownership:

chown: Changes the owner and/or group of files

chgrp: Changes only the group ownership

Syntax:

chown user:group file
chown user file
chgrp group file

-R flag: Apply recursively to directories and their contents

Tip: You can use UIDs and GIDs instead of names (e.g., 1000:1000)

About ACLs:

ACL (Access Control List): Extends standard permissions to grant specific access to additional users/groups

Entity Types:

  • u: User permissions
  • g: Group permissions
  • o: Other permissions
  • m: Mask (limits effective permissions)

Default ACL: Applies to newly created files in a directory

Note: Requires ACL support on the filesystem (ext3/4, XFS, etc.)

About Umask:

Umask: Determines default permissions for newly created files and directories

How it works:

  • Files: 666 - umask
  • Directories: 777 - umask

Common Values:

  • 0022: Standard (644/755)
  • 0002: Group-friendly (664/775)
  • 0077: Private (600/700)

Persistence: Add to ~/.bashrc or ~/.profile to make permanent

About File Attributes:

chattr: Change file attributes beyond standard permissions

lsattr: List current file attributes

Most Used Attributes:

  • +i (Immutable): File cannot be modified, deleted, or renamed - even by root!
  • +a (Append): Data can only be added, not removed. Perfect for log files.
  • +A (No Atime): Don't update access time. Improves disk performance.

‚ö†Ô∏è Important:

Requires root or CAP_LINUX_IMMUTABLE capability. Use with caution!

About Sudoers:

sudo: Execute commands as another user (usually root)

visudo: Safe editor for sudoers files with syntax validation

Common Use Cases:

  • Service Management: Let users restart specific services without full root
  • Package Updates: Allow developers to run apt/yum commands
  • Docker/K8s: Grant container management permissions

  • Avoid NOPASSWD for sensitive commands
  • Place custom rules in /etc/sudoers.d/
